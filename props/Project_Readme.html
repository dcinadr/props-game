Note: To use this template you need to install:

  • NPM Task Runner extension
  • TypeScript 2.0.6 editor
  • Microsoft ASP.NET and Web Tools Preview 2

Step 1: NPM Restore

Now that your project has been created, you need to wait until all npm packages have been restored. This process takes a while depending on your machine and your internet connection speed. Now may be a good time to get a cup of coffee.

In the Dependencies node you should see a "Restoring Packages.." message. If you don't, you can always Right Click the Dependencies node and click on Restore Packages.

When the Package Restore has finished, you will see a message in the solution explorer that says "Dependencies - not installed". This is a known issue, but rest assured you'll now be able to use Ionic.

If you have any problem with this process, you can look for any error in the Output Window (Ctrl+Alt+O), under the Npm/Bower category.

Step 2: Enable Live Reload for Ionic

Ionic uses NPM scripts to compile your applications TypeScript and SASS files into JavaScript and CSS, and places the output into the www folder. By default, Visual Studio is using an ionic:build script when you build your project.

While developing your app using the Ripple emulator, you may want to setup your project to live reload changes into the browser as you work on your files. To do this:

  1. Open the Task Runner Explorer from the Menu: View->Other Windows-Task Runner Explorer
  2. Right click the ionic:build task and choose the Bindings | Project Open menu to setup the ionic:build task to run when you load your project.
  3. Run your app in the Ripple browser.

After following the above steps, you'll now be able to make edits to the files in your src folder and your changes will automatically show up in the Ripple emulator.
